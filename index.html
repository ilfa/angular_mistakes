Angular, any problems?

Change detection

  • Your application is big
  • You have data calculation on render
  • You listen sockets in Angular Zone

Default change detection

  • Each zone update
  • Checks all tree from root

OnPush change detection

OnPush change detection

  • Each zone update
  • Don't go down if node clear
  • But event marks subtree up to root

How fix sockets problem?

  • Use OnPush strategy
  • Subscribe on data update out of zone

How control CD manually?

  • markForCheck - for mark subtree up to root
  • detectChanges - for trigger CD
  • detach - for skip CD cycles
  • reattach - cancel detach
